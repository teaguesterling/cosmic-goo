cosmic-goo

Grammar Of Operations — a noun → verb → object layer for the COSMIC desktop

Pick a subject (selection, file, URL, app, anything a plugin emits), apply a verb, optionally an object and adverbs. One engine (a Rust port of the original shell engine, TOML plugins) drives the CLI today — with a type-aware negotiation planner and sigil-less inference underneath — and, by design, the launcher and a compose dialog next. This page is a live map of what exists and where it's going.

The grammar

Every interaction is a little sentence. The same backend resolves it whether you type it, bind it to a key, or build it in a picker.

# subject        verb        object/adverbs
goo summarize ./notes.md            # file → text verb
goo activate :app:firefox          # source item → handle verb
goo move-to :app:Claude :ws:2     # two-step (verb takes an object)
goo think "recursion" --depth=ultra  # adverb modulates the verb
echo hi | goo critique --via=clipboard  # stdin subject, routed output

Addressing — how a subject is written

Files and URLs need no sigil (native feel). Everything rewrites to one canonical goo:// URI, then dispatches.

^ / custom sigilsingle-char shorthand → a core form (^→+clip:; @ free for you)goo summarize ^
goo://… / goo+…the canonical URI directly (scripts / IPC / launcher; goo:// is xdg-registrable)goo summarize 'goo+file:///abs/x.md'

Core sigils: :source:input → goo://source/input, +scheme:value → goo+scheme:value. Everything else is a customizable [[sigils]] — one char that expands into a core form. When no subject is given, it falls back: stdin → PRIMARY selection → clipboard → focused app.

Anatomy of a plugin

Plugins are pure TOML declaring types / sources / verbs / adverbs / sigils / aliases / dispatch. The engine merges them (last-wins by name) and shells out to render commands.

# a verb is the unit of action
[[verbs]]
name        = "move-to"
accepts     = ["application/vnd.cos-cli.app"]   # subject type(s)
object_type = "application/vnd.cos-cli.workspace" # takes an object →
object_source = "workspaces"                    # …drawn from this source
cmd = "cos-cli move -i {subject.metadata.index} -w {object.metadata.index}"

Frontends — now & next

The CLI is the canonical entry point. The picker dialog exists in v0; the launcher and a native three-pane GUI are roadmap. All speak the same goo:// grammar.

Roadmap

The engine, the type-system / negotiation layer, and the whole sigil-less inference arc are done. The next major arc is the native compose-GUI; the launcher and daemon follow it by design.
