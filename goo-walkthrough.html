A session with goo

A deploy just failed. You reach for cosmic-goo — the noun → verb → object grammar for the COSMIC desktop. Below is the whole session: every command run against the real engine, every output captured verbatim. Then the UX writeup — why it feels the way it does.

Captured 2026-06-01 from crates/target/release/goo · the web-1 / db-1 / api-gateway servers are a tiny demo plugin (safe echo verbs); the stop ×3, --explain, and what examples are the real shipped registry.

Act IYou have text. Say something to it.

The verb comes first (Unix muscle memory: goo verb subject --flag). A bare string is just the subject — typed as text/plain, no ceremony.

goo upper
$ goo upper "deploy failed: connection refused"
DEPLOY FAILED: CONNECTION REFUSED
noun → verb. upper is the verb; the quoted string is the subject. No flags, no mode — the verb declares it accepts: text/*, the string fits, it runs.
goo base64-encode
$ goo base64-encode "secret"
c2VjcmV0
No subject given? goo falls back, in order: stdin → PRIMARY selection → clipboard → focused app. So echo secret | goo base64-encode works too — the pipe becomes the subject.

Act IIWhat can I even do here?

The registry is a vocabulary. You can ask it what it accepts — and the answer is the same projection that drives tab-completion and the (coming) GUI. One source of truth, three surfaces.

goo what =text/markdown
$ goo what =text/markdown
applicable verbs for =text/markdown  (type: text/markdown)
    calc          Evaluate a mathematical expression
    notify        Send a desktop notification with the selected text
    qr-encode     Render the text as a QR code in the terminal
    upper         Convert to UPPERCASE
    lower         Convert to lowercase
    wc            Word, line, and character counts
    base64-encode Encode as base64
    url-encode    Percent-encode for use in a URL
    ...
"What can I do with this?" =text/markdown names a type directly. Every verb whose accepts admits markdown shows up — subtype-aware (markdown is text).
goo describe stop
$ goo describe stop
stop  [!] ×3
  [containers]  Stop the container   accepts: application/vnd.container      confirm: true
  [media]       Stop playback        accepts: (no subject)
  [services]    Stop the unit        accepts: application/vnd.systemd.unit   confirm: true
One word, three meanings. stop is polymorphic — three plugins ship it, each for a different subject type. goo stop :ctr/web hits the container impl; bare goo stop hits media. The ×3 is the registry telling you how polymorphic the verb is.

Act IIIAct on things by name. — the keystone

You shouldn't have to remember addresses. Type the name of a thing; goo scores it across your sources and answers with a confidence band — and the band decides how loud it is.

goo web-1
$ goo web-1
ping web-1 (10.0.0.11) ... reachable
DEFINITIVE Bare name, no sigil, no verb. web-1 is an exact, unique server id — so goo resolves it silently and runs the type's default verb (ping). DEFINITIVE is the only band that fires silently everywhere, scripts included — because an exact-unique match can't surprise you.
goo gateway (at a terminal)
$ goo gateway   # the server's title is "api-gateway (production, us-east)"
goo: inferred 'gateway' → :srv/api-gateway  (band: HIGH; use the explicit form to suppress)
ping api-gateway (10.0.0.30) ... reachable
HIGH Not exact, but one clearly-dominant match — gateway is a whole word inside that long, descriptive title. goo resolves it and teaches you the precise form (:srv/api-gateway) on stderr. (Scoring ratios against the matched word, not the whole title, so a descriptive name doesn't sink a clear match out of HIGH — see Sharp edges.)
goo web (at a terminal)
$ goo web
goo: 'web' is ambiguous — pick one:
  1) :srv/web-1                        web-1 (prod)
  2) :srv/web-2                        web-2 (staging)
re-run with the explicit address; suppress this list with GOO_INFER_NO_NUDGE=1.
[exit 2 — nothing ran]
MEDIUM Two servers match "web". goo won't guess — it shows a picker and runs nothing. You re-run with the address you see. Ambiguity is surfaced, never resolved by luck.
goo gateway (in a script / pipe — non-TTY)
$ goo gateway   # stdout is a pipe ⇒ script context
goo: would have inferred 'gateway' → :srv/api-gateway (HIGH band) — not auto-resolving
     in script context. Use the explicit form, or GOO_INFER_STRICTNESS=tty.
goo: unknown verb or subcommand: gateway
[exit 1 — fell through, nothing ran]
The safety property. The same HIGH match that resolved at your terminal refuses to auto-resolve in a script. In pipes / CI / cron, only DEFINITIVE fires silently; everything fuzzier logs what it would have done and falls through. No fuzzy guess can ever surprise automation.
goo ping web (verb-aware)
$ goo ping web
goo: 'web' is ambiguous — pick one:
  1) :srv/web-1                        web-1 (prod)
  2) :srv/web-2                        web-2 (staging)
[exit 2]
Verb-aware bias. Once a verb is in play, inference only scores sources that verb accepts. ping takes servers, so only servers are considered — connect fox would look at hosts/devices, summarize fox at text. It narrows the pool, the relevance, and the privacy surface.

Act IVDangerous things look dangerous.

The risk of a verb is part of its grammar, surfaced the same way on every surface.

goo what :srv/web-1
$ goo what :srv/web-1
applicable verbs for :srv/web-1  (type: application/vnd.demo.server)
    ping              Check a server is reachable
    restart  [!]      Restart the service on a server
    destroy  [!!]     Tear down a server
Chips. [!] = wants confirmation, [!!] = destructive. One chip vocabulary, cited by every surface (CLI, describe, what, the future GUI) — invent a glyph and it's a bug.
goo destroy db-1
$ goo destroy db-1
goo: about to Tear down a server [!!]
       subject: db-1
       runs: printf 'destroyed %s\n' 'db-1'
     proceed? [y/N] _   # cancelled here (exit 130)
Confirm gates before the deed. A confirm/destructive verb leads with its description and chip, names the subject, shows the exact command on the runs: line, and waits for y/N. A script can pre-approve a specific verb with --confirm-dangerous=destroy — scoped, per-invocation, and never silent (it prints an auto-approve note); anything else still prompts.

Act VWhen types don't line up, goo finds a path.

You ask for an outcome; the engine negotiates a route. A two-layer planner coerces the input and picks the delivery surface — and --explain shows its work.

goo --explain summarize =text/markdown
$ goo --explain summarize =text/markdown
Accept: application/vnd.goo.surface  */*   · caps {display}
subject: text/markdown (via explicit)
text/markdown → (summarize) → text/plain → cosmic-edit → application/vnd.wayland.surface   (cost 8)
  commands (in run order):
    cosmic-edit {in.path|q}
The planner negotiates. "Summarize markdown, for my display." goo plans the route markdown → summarize → text → cosmic-edit → a surface (cost 8) and prints the exact command each hop runs. No guessing what tool will open — you can see it before it happens.
goo --explain view =image/png --paths
$ goo --explain view =image/png --paths
4 route(s) from image/png (≤3 hops/layer, most-preferred first):

  1. cost  4 → application/vnd.wayland.surface
       image/png → (view) → image/png → eog → surface
  2. cost 24 → application/vnd.wayland.surface
       image/png → (view) → image/png → chafa (lossy) → text/x-ansi → cosmic-edit → surface
  3. cost  0 → image/png
       image/png → (view) → image/png
  4. cost 16 → text/x-ansi
       image/png → (view) → image/png → chafa (lossy) → text/x-ansi
Many roads, ranked. There's more than one way to view a PNG: eog (cheap, cost 4) or chafa straight in the terminal (lossy, cost 24). The planner enumerates and cost-ranks them; pin the one you want with --using, or send the bytes somewhere with --to.

The walkthrough showed what happens. Here's why it's shaped this way — eight principles, each pinned to a moment you just saw.

seen in · Act III

1. Bare typing is the default. Sigils are how you're precise.

Today most launchers make you pick a mode. goo inverts it: firefox should work as well as :app/firefox. The sigil isn't the price of entry — it's the tool you reach for to disambiguate when a bare name isn't enough. Inference at the address layer is what lets bare names resolve through the registry; the sigil is always there when you want to be exact.

seen in · Act III

2. Confidence bands, not a threshold.

A single cutoff ("resolve if score > N") forces one behaviour on every match. Bands let the same match mean different things in different places — the band is the user-facing model; the response adapts to context.

BandAt a terminalIn a script / pipe
DEFINITIVEresolve silentlyresolve silently (exact, unique — safe)
HIGHresolve + one-line nudgelog "would have" → fall through
MEDIUMpicker, run nothingfall through
LOWfall through to textfall through to text
seen in · Act III (the script run)

3. Determinism for automation is non-negotiable.

Scripts, cron, and CI get exactly one safe class — DEFINITIVE. Everything fuzzier degrades to a stderr log and a fall-through. The blast radius of "goo guessed wrong" is bounded to the interactive terminal, where a human is watching. GOO_INFER_STRICTNESS=tty opts a script into the looser behaviour, explicitly.

seen in · Act II & IV

4. One projection, every surface (a single source of truth).

goo what, tab-completion, the chips, and the future compose-GUI all read the same OPTIONS projection of the registry. A consumer can't invent its own ranking or its own glyph without it being a bug. The walkthrough's what output and a <TAB> menu are the same answer rendered two ways.

seen in · Act I & the whole session

5. The grammar: noun → verb → object → adverbs.

Every interaction is a little sentence. The CLI is verb-first (Unix muscle memory). The compose-GUI and the planned launcher are noun-first (Spotlight muscle memory: pick the thing, then the action). Both speak to the same engine — the surface matches the grammar you're already typing, not the other way around.

seen in · Act II

6. Polymorphism is a feature, surfaced honestly.

stop meaning three different things isn't a name collision to resolve — it's the language being expressive. The registry keeps each impl by (name, accepts) and dispatches by the subject's type. The ×N count tells you, up front, how many meanings a word carries.

seen in · Act IV

7. Dangerous looks dangerous — before you commit.

Risk is declared in the verb's TOML (confirm, destructive) and rendered as a fixed chip vocabulary: [!], [!!]. The chip warns; the confirm gate shows the exact command and waits. You always see the blade before it swings.

seen in · Act V

8. Types negotiate; they don't just fail.

When a subject's type isn't what a verb wants, goo doesn't 415-and-quit — it plans a route through known converters, cost-ranked, and tells you the path (or, if there genuinely isn't one, the exact tool to install). The negotiation planner, output routing (--as / --to / --using), and the teaching --explain all fall out of one algorithm.

Sharp edges — what the demo glossed over

This section is here on purpose. A walkthrough that shows only wins is a sales pitch; the honest version names where it bites today. Several of these were surfaced by an adversarial review of this very page — which is exactly why every artifact like this should carry one.

⚠ Caching is mtime-watched, not a live watcher

On the one-shot CLI, a source caches only if it declares a watch file (today just recent); command/dbus sources (apps, bluetooth) recompute every run until the good daemon adds inotify/dbus. goo reload forces a refresh. The cache is never stale by mtime — but a same-second edit on a coarse-mtime filesystem could slip through (desktop filesystems are nanosecond).

⚠ ssh Include isn't followed

The recent-files watch is exact, but watching ~/.ssh/config alone would under-cover Include / ~/.ssh/config.d/ — scope any "ssh config changes invalidate" claim accordingly.

⚠ The negotiation path doesn't confirm yet

A destructive verb reached via type-coercion/negotiation currently skips the y/N gate — only the plain-command path confirms. Plain verbs (the common case) are gated; the negotiated path is a tracked follow-up.

⚠ Bands are calibrated, not proven

The scoring is a heuristic. HIGH was effectively unreachable for long, descriptive titles until a fix this week (it ratio'd against the whole title, not the matched word). The thresholds are tuned to fixtures, not derived — treat the band a query lands in as "usually right," not guaranteed.

⚠ The picker is display-only; polymorphism biases on one impl

The MEDIUM picker prints addresses to re-type — there's no interactive number-pick yet, and in verb-position its "re-run with the explicit address" hint is verb-agnostic. And verb-aware inference scores against one impl's accepts, not the union across a polymorphic verb's targets.

⚠ The demo used safe stand-ins

The web-1 / db-1 / api-gateway servers were a labelled echo plugin so nothing fired for real. Real sources need their backing tools (cos-cli, systemctl, bluez); the registry features (stop ×3, --explain, what) are genuine, but a fresh machine won't have every source populated.

The shape of it

Strip away the features and what's left is a wager: that a desktop is more usable when speaking to it is as fast as typing, when the system is honest about ambiguity and risk, and when the same little grammar works at a prompt, on a keybind, and in a picker. The walkthrough is that wager, paid out one real command at a time.
