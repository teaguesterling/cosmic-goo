What goo addresses today

Every subject resolves to one canonical goo://<domain>/<path> URI. Two kinds of domain: value-domains (built-in, no enumeration) and source-domains (plugin-declared, enumerable via list_cmd).

What's possible in each area

Click each category. Has = what goo ships now. Could = plugin sketches (each is a [[sources]] / [[verbs]] TOML — no engine work). How others do it = reference points across the launcher ecosystem.

📁 Files & filesystem find, open, recent, trash, mime-aware actions has 2 · could add 5
Has
:filefiles.toml — find by name openxdg-open default
Could add
:recent~/.local/share/recently-used.xbel :trashgio trash list-trash :mountudisksctl status / lsblk -J :bookmark~/.config/gtk-3.0/bookmarks previewkitty +icat / qlmanage analogue
How others do it

Spotlight indexes the FS continuously and exposes recent + mime-aware actions. KRunner has locations for paths plus recentdocuments. cosmic-launcher / pop-launcher have files (find) + recent. Raycast layers per-mime "actions panel" — open, reveal, move, tag — which goo can do natively via verb dispatch on inode/file + structured-suffix subtypes.

🔊 Audio & media sinks, sources, streams, MPRIS players has 2 · could add 3

Prioritized gaps

Most "missing" items above are a [[sources]] TOML with a list_cmd — no engine changes. That's the architectural payoff: gaps close at plugin speed.

Easy wins · ~20–30 line TOML each

  • :recent~/.local/share/recently-used.xbel — every GTK/Qt app updates it
  • :emojiUnicode CLDR JSON + copy default verb
  • :mountudisksctl status / lsblk -J
  • :hist-sh~/.bash_history / ~/.zsh_history / fish history
  • brightnessbrightnessctl --machine-readable
  • :mprisplayerctl --list-all → per-player verbs
  • :pr / :issuegh pr list --json / glab

Medium effort · existing tools, light parsing

  • :bookmarkFirefox places.sqlite (sqlite3 query in list_cmd)
  • :tabper-browser RPC (qutebrowser has it; Firefox via extension)
  • :snip & :notefile-backed; the planned ^name buffers are the natural anchor
  • :pass + :totppassword-store + pass-otp; rofi-pass-style flow
  • units / translateverbs over units / trans / argos-translate
  • :image / :k8sdocker / kubectl JSON output

Bigger lifts · real integration work

  • :win per-window switcherneeds cosmic-comp APIs (parallels :app as :win)
  • :cal / :contact / :mailkhal / khard / notmuch — the productivity-launcher tier
  • cosmic-config bridgeread/write the desktop's real settings: scheme, wallpaper, panel
  • KDE Connectcross-device send/recv/control

What goo uniquely covers

Worth not losing sight of — categories where goo already leads, or has infrastructure no other launcher has.

  • Containers — :ctr ships first-class; no major launcher does
  • SSH hosts — :ssh ships first-class; ulauncher has it only as a third-party extension
  • tmux sessions — :tmux ships; effectively unique
  • Git branches & repos — :br / :repo; Raycast has only via paid extension
  • systemd services — :svc; KRunner has it; gnome-ecosystem doesn't
  • Processes (kill, signal) — :ps; partial in KRunner, ulauncher
  • Type-virtual sigil — =text/markdown; protocol-level, nothing else has this
  • OPTIONS discovery surface — goo options; one composable JSON the compose-gui, shell completion, and (future) daemon all share
  • Verb-on-noun sentence grammar — gnome-do's DNA, with type-driven applicability and channel-based coercion
  • Output routing — --to/-o routes any result to file/clipboard; the only launcher that treats where the answer goes as a first-class slot
  • Screenshot · OCR · QR scan — screenshot, capture-region, capture-file, ocr-image, ocr-region, scan-qr, scan-qr-image all ship as first-class verbs (grim/slurp/tesseract/zbarimg); other launchers leave these to standalone tools
